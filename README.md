## Git 

i. main 為大家主要支線，請勿隨便亂push

i. 下載專案後，如要開發新功能 or 重大更改，請先創一個自己的brantch開發

ii. 當你要Push到雲端前，請先確認一下動作:

    1. 確認專案各項功能正常
    2. Fetch新的資料，確認其他人有無更新資料: 
        a. 若無更新，直接Push即可
        b. 若有更新，請先將最新傳案Fetch下來在'地端'merge，測試無誤後才可以push
